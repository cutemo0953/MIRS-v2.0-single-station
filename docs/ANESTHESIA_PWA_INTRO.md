# 麻醉 PWA 簡介 - 徵求麻醉科專家意見

**專案：** xIRS (野戰醫療資訊系統) - MIRS 麻醉模組
**版本：** v1.5.1
**狀態：** Phase A/B 完成，尋求領域專家回饋

---

## 一、專案背景

在災難/戰時的野戰手術站 (BORP - Backup Operating Room Point)，我們需要一套能夠：
- **完全離線運作** 72 小時以上
- **記錄完整麻醉過程**（生命徵象、用藥、氣道管理）
- **管制藥品帳目平衡**（雙人簽核、絕不漏記）
- **追蹤氧氣瓶消耗**（自動估算剩餘時間）
- **支援戰場快速模式**（5 個決策點，30 秒完成評估）

---

## 二、目前已完成功能

### 2.1 案例管理 ✅
- 從 CIRS 掛號系統選取病患，或掃描傷票 QR Code
- 斷網時可建立暫存病患 (TempRegistration)
- 連線恢復後自動對齊身份

### 2.2 事件流記錄 ✅
- **Append-Only 設計**：所有記錄只能新增、不能修改（符合醫療法規）
- **生命徵象**：血壓、心率、SpO2、EtCO2、體溫、氧氣流量
- **用藥記錄**：10 種常用藥物一鍵輸入
- **氣道事件**：插管/拔管記錄（含 Cormack-Lehane 分級）
- **里程碑**：麻醉開始、切皮、關皮、麻醉結束

### 2.3 管制藥品帳冊 ✅
- **雙重登錄會計**：`Balance = DISPENSE - (ADMIN + WASTE + RETURN)`
- **廢棄需見證**：WASTE 交易必須有第二人 PIN 簽核
- **結案阻擋**：Balance ≠ 0 無法關案
- **完整交易歷史**：每筆交易記錄時間、執行者、見證者

### 2.4 氧氣瓶認領 ✅
- **Claim 模型**：案例綁定氧氣瓶，防止重複使用
- **自動估算**：根據充填百分比和流量計算剩餘分鐘
- **關案釋放**：案例結束自動解除綁定

### 2.5 術前評估 ✅
- **標準模式**：ASA 分級、NPO 狀態、Mallampati 分數、完整病史
- **戰場模式**：5 個快速決策點（Airway/HD/NPO/Bleeding/Duration）

## 三、尋求回饋的問題

### 🎯 問題 1：術前評估流程
目前設計：護士可填寫，但核准需要醫師 PIN。
**疑問：** 戰場情境下，醫師可能同時處理多台手術，這個流程是否太繁瑣？

### 🎯 問題 2：生命徵象記錄頻率
目前設計：預設每 5 分鐘記錄一次。
**疑問：** 在戰場環境下，這個頻率是否合理？需要更頻繁還是可以更彈性？

### 🎯 問題 3：管制藥品流程
目前設計：廢棄必須雙人簽核，結案前 Balance 必須為 0。
**疑問：**
- 戰場情境下，可能只有一人在場，如何處理？
- 「離線存證 + 事後補簽」的設計是否可接受？

### 🎯 問題 4：PACU 流程（待開發）
計畫設計：Aldrete 評分、出院標準。
**疑問：** 野戰環境的 PACU 標準是否需要簡化？

### 🎯 問題 5：角色切換機制
目前設計：護士/醫師共用同一 PWA，醫師操作需 PIN 提升權限。
**疑問：** 這個設計在實際操作中是否順暢？
